#### hash模式
1. 任何情况下都能做前端路由
2. 作用
```
baidu.com/#1 展示1内容
baidu.com/#2 展示2内容
```
3. 缺陷:SEO不友好
谷歌浏览器会认为
```
baidu.com/#1
baidu.com/#2
```
等效于
```
baidu.com
```
也就是说谷歌浏览器会把#后面的内容吃掉,导致服务器压根儿收不到hash值(除非对服务器做特殊配置)

#### history模式
1. 只有一种情况下能做前端路由:后端将所有前端路由都渲染到同一个页面(比如首页，不能是404)
2. IE8以下不支持history模式
3. 
```
<a href="#1">go to 1</a> // hash模式
<a href="/1">go to 1</a> // history模式
```
